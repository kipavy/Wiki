# Architectural Patterns

Recurring "shapes" that web apps tend to follow. You don't have to invent these — you pick the ones that fit your project. Each page gives you a verdict up front, a diagram, and when to use it.

* [SPA vs. MPA](spa-vs.-mpa.md) — how pages load and navigate.
* [Rendering Strategies](rendering-strategies.md) — where the HTML gets built (CSR / SSR / SSG).
* [Monolith vs. Microservices](monolith-vs.-microservices.md) — how you split (or don't split) your codebase.
* [Communication](communication.md) — how services talk: synchronous vs. event-driven.
* [MVC](mvc.md) — the classic way to organize code inside the backend.
